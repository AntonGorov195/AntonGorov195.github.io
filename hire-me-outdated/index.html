  • I Value High Quality Software

    Code quality matters. But what is high quality code? You may ask. Here are the characteristics of good code:

    1. Error Handling

      I've seen it too many times. My peers ignoring error handling and it always bites them in the ass. Error handling is not an afterthought, it is an integral part of software design. When starting a project I always ask myself how would I handle errors. Not once I came to the conclusion that designing with error handling was a mistake.

    2. Takes Advantages of Tooling

      In the world of programming, there are so many tools at your disposal. Version control (git), dev tools, debuggers, hot reloading should be part of your workflow. These tools should be mastered, and used to the fullest extent. One of my gripes in university is that we never went deep into this subject (besides git). However, I value their power and ready to use them as nessasary.

    3. Written With Tests in Mind

      Writing test at the start of the project can be rough. Due to constantly changing requirements, the test may quickly become obsolete. However, if the code was written improperly, it can be tough to write them. The right way to code is to have a clear strategy when it comes to tests. Sometimes, writing tests can be straightforward.

    Here is how those principles applied to my platforming game.

    1. Initialize a git repo (2)
    2. Choose what technologies to use, I pick the Odin programming language + Raylib (2)
    3. Choose an error handling strategy, I chose C-style error codes (1)
    4. Learn how to used the debugger and hot reloading in Odin (2)
    5. Have all the game data be a simple array of bytes (3)
    6. Segregate side effects (such as user input) into a seperate chunk of memory (3)
    7. Make a relay system, where you can record the initial state and user inputs (3)
    8. Save those replays into a file, this way you can make automated test (3)
    9. Write good error messages for cases where the replay fails (1)
